- **统一 LLM 路由层**：`llm/LLMClientFactory` 作为核心网关，根据模型前缀或供应商 ID 动态路由至 Gemini、OpenAI 兼容及 Ollama 等客户端，屏蔽底层 API 差异。
- **多模态任务驱动工厂**：`task/pipeline_drivers` 与 `visual_drivers` 采用工厂模式，将数十种外部 AI 服务（如 RunningHub、Kling、Sora）封装为统一驱动接口，由 `PipelineProcessor` 进行异步提交与状态轮询。
- **智能体技能与 SOP 动态注入**：`agents/skill_loader` 解析 Markdown 格式的 SOP 与技能定义，支持在运行时动态构建索引并注入到 `script_writer_core` 的智能体 System Prompt 中，实现业务流程的可配置化。
- **分层执行与算力风控**：`script_writer_core` 负责高层逻辑编排与工具调用，`task/sync_task_executor` 利用进程池隔离耗时同步请求；全链路集成 `check_computing_power` 机制，在关键节点校验用户算力余额以保障系统资源安全。
