- **基础设施层**：`database.py` 封装 `pymysql` 连接池逻辑，提供 `get_db_connection` 上下文管理器及 `transaction` 事务控制，确保资源自动释放。
- **领域模型层**：采用“数据类 + 静态服务类”模式（如 `User`/`UsersModel`），每个业务实体对应独立模块，内部包含 SQL 定义与 CRUD 方法。
- **演化机制**：通过 `sql/migrations/` 目录下的时间戳命名 SQL 脚本管理 schema 变更，并由 `migration.py` 集成 Alembic 实现启动时自动升级。
- **导出边界**：`__init__.py` 统一暴露核心 Model 类与数据库工具函数，屏蔽底层文件结构。
