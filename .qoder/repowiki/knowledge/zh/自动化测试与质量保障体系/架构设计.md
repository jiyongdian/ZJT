- **分层架构**：采用 Page Object 模式（`helpers/page_objects.py`）封装页面交互逻辑，实现 UI 元素定位与业务操作的解耦。
- **测试执行层**：基于 `pytest` 组织测试用例，通过 `conftest.py` 管理全局 fixtures（如浏览器上下文、认证 Token、API 客户端），支持会话级复用以提升执行效率。
- **数据驱动与 Mock**：在 E2E 测试中广泛使用 `page.route()` 拦截关键 API（如算力查询、解析接口）并返回 Mock 数据，确保测试环境的稳定性与独立性；单元测试则通过 `config_unit.yml` 隔离测试数据库配置。
- **进度管理工具**：内置 `test_navigator.py` 命令行工具，解析 `test_sessions` 下的 JSON 状态文件，动态计算模块/功能/步骤的完成度，并提供“下一个待测项”指引，解决大模型上下文受限时的测试断点续传问题。
- **辅助工具集**：提供 `helpers/editor_helpers.py` 等专用脚本，通过 JS 注入绕过前端框架（如 Vue）的异步渲染或遮挡问题，增强自动化操作的鲁棒性。
