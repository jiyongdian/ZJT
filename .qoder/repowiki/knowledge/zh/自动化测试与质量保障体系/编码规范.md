- 所有 E2E 测试用例必须通过 `page.route()` 或 fixture 预置方式 Mock 外部依赖接口（如 `/api/user/computing_power`），防止因网络波动或权限校验导致测试重定向或失败。
- 页面交互优先使用 JS 注入（`page.evaluate`）或直接操作 DOM 属性，以规避前端框架（如 Vue v-model）的事件绑定延迟或元素遮挡导致的 Playwright 原生操作失败。
- 测试数据清理遵循“后置清理”原则，在 fixture 的 `yield` 后通过 `api_client.delete` 或 SQL 语句删除创建的临时资源（如世界、工作流、会话），确保测试环境的幂等性。
- 认证信息通过 `context.add_init_script` 在页面加载前注入 `localStorage`，避免在每个测试用例中重复执行 UI 登录流程，显著提升测试执行速度。
