# 营销图片生成流程

## 适用场景
用户需要生成或编辑营销图片，包括但不限于：
- 商品展示图 / 产品图
- 广告海报 / 宣传图
- 社交媒体配图（小红书、抖音封面、朋友圈等）
- 品牌视觉素材
- **图片编辑**：基于用户提供的原始图片进行风格转换、背景替换、元素修改等

## 流程步骤

### 步骤1：收集需求信息

通过 `ask_user` 工具向用户确认以下信息（如果用户描述中已包含则跳过）：

1. **图片用途**：商品展示？海报？社交媒体配图？图片编辑修改？
2. **是否有原始图片**：用户是否提供了需要编辑的原始图片？
3. **主体内容**：展示什么商品/场景/人物？编辑指令是什么？
4. **风格偏好**：写实？插画？3D渲染？扁平化？
5. **尺寸比例**：1:1（Instagram）、4:3（小红书）、16:9（横版海报）、9:16（竖版/手机）
6. **其他要求**：颜色倾向、氛围、参考图等

**示例**：
```
ask_user(
    question="请确认图片用途和风格偏好：",
    options=["商品白底图", "商品场景图", "社交媒体配图", "广告海报"]
)
```

### 步骤2：委托给营销图片专家

收集完需求后，调用 `call_agent` 将任务委托给营销图片专家 `marketing-image`。

**文生图场景**（用户没有提供原始图片）：
```
call_agent(
    AgentName="marketing-image",
    task_description="请为用户生成一张营销图片，需求如下：
    - 用途：[图片用途]
    - 主体：[主体内容]
    - 风格：[风格偏好]
    - 比例：[从用户偏好中读取]
    - 分辨率：[从用户偏好中读取]
    - 其他要求：[额外要求]

    ⚠️ 重要：比例和分辨率是用户在界面上明确设置的，已自动注入系统，你不需要手动指定 aspect_ratio 和 image_size 参数（系统会自动强制校验）。只有在用户未设置或设置为'auto'时，你才可以自行决定。
    请先检查模型和算力，然后构建提示词并生成图片。",
    user_input="[用户的原始描述]"
)
```

**图片编辑场景**（用户提供了原始图片）：
```
call_agent(
    AgentName="marketing-image",
    task_description="请为用户编辑图片，需求如下：
    - 编辑指令：[用户希望如何修改图片]
    - 比例：[从用户偏好中读取]
    - 分辨率：[从用户偏好中读取]

    ⚠️ 重要：比例和分辨率是用户在界面上明确设置的，已自动注入系统，你不需要手动指定 aspect_ratio 和 image_size 参数（系统会自动强制校验）。只有在用户未设置或设置为'auto'时，你才可以自行决定。
    请使用 edit_image 工具进行图片编辑。",
    user_input="[用户的原始描述]"
)
```

### 步骤3：确认专家返回并继续

专家 `marketing-image` 返回工作总结后，包含：
- `project_ids`：已提交的图片生成任务 ID
- 使用的模型名称
- 消耗的算力
- 或者失败原因

**注意**：
- 专家返回工作总结即表示任务已提交，你不需要等待图片生成完成
- 前端会自动轮询图片生成进度并在完成后展示结果
- 你不需要向用户报告"图片正在生成中"或"请稍等"——前端会处理进度展示

### 步骤4：进入下一步或结束

专家成功返回后：
- 如果用户还有其他需求（如继续生成更多图片、生成视频等），继续执行
- 如果当前任务已完成，直接结束，不需要额外输出关于图片状态的总结
- **严禁**说"图片正在生成中，请稍等"、"我会在完成后告知您"之类的话

如果专家返回失败：
- 告知失败原因
- 询问用户是否需要重试或换一种方式

### 步骤5：迭代优化（如用户需要）

如果用户对结果不满意：
1. 收集用户的修改意见
2. 再次调用 `call_agent` 生成新图片，在 `task_description` 中包含修改要求
3. 重复步骤3-4

## ⚠️ 注意事项

1. **不要自己调用生图工具**：生图操作全部委托给 `marketing-image` 专家
2. **需求收集要充分**：确保在委托前了解用户的完整需求
3. **尊重用户偏好**：如果用户消息中包含 `[用户图片偏好]`（如比例、分辨率、模型），必须在 `task_description` 中明确传递给专家，不要忽略
4. **算力提醒**：如果专家报告算力不足，需要明确告知用户
5. **不要重复报告进度**：专家返回工作总结后，你不需要向用户说"图片正在生成中"或"请等待"——前端会自动轮询并展示结果
6. **不要展示图片**：前端会自动在图片完成后展示结果，你不需要在回复中用 Markdown 语法插入图片或描述图片内容
