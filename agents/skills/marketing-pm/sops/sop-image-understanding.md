# 图片理解流程

## 适用场景
用户上传了图片并希望：
- 识别图片内容并反推可用于重新生成的提示词
- 分析图片的艺术风格（色调、构图、风格流派等）
- 对比多张图片的相似度和差异
- 识别图片中的文字（OCR）

## 流程步骤

### 步骤1：确认分析需求

通过 `ask_user` 工具确认用户希望进行哪种分析（如果用户描述中已明确则跳过）：

```
ask_user(
    question="您希望对这张图片进行什么分析？",
    options=["识别内容 + 反推提示词", "风格分析", "图片对比", "文字识别（OCR）"]
)
```

**注意**：
- 如果用户只提供了 1 张图片但选择了"图片对比"，提醒用户需要上传至少 2 张图片
- 如果用户没有明确指定，默认执行"识别内容 + 反推提示词"

### 步骤2：委托给图片理解专家

收集完需求后，调用 `call_agent` 将任务委托给图片理解专家 `image-understanding`。

```
call_agent(
    AgentName="image-understanding",
    task_description="请分析用户提供的图片，需求如下：
    - 分析类型：[识别内容/风格分析/图片对比/文字识别]
    - 用户补充要求：[如有]

    请根据分析类型执行对应的分析，并输出结构化的分析结果。
    如果是识别内容类型，请额外生成可用于文生图的英文提示词。",
    user_input="[用户的原始描述]"
)
```

### 步骤3：确认专家返回并继续

专家 `image-understanding` 返回分析结果后：
- 将分析结果呈现给用户
- 询问用户是否需要基于分析结果执行后续操作（如用提示词生成新图片）

### 步骤4：后续操作（可选）

根据分析结果，用户可能希望：
1. **用反推的提示词生成新图片** → 切换到 `sop-image-generation` 流程，将提示词传递给 `marketing-image` 专家
2. **进一步分析** → 继续让 `image-understanding` 专家分析
3. **结束** → 完成当前任务

## ⚠️ 注意事项

1. **不要自己分析图片**：图片理解操作全部委托给 `image-understanding` 专家
2. **图片必须存在**：用户必须已上传图片才能触发此流程，如果用户只说了"分析图片"但没有上传，应提示用户先上传
3. **分析结果可复用**：反推的提示词可直接传递给 `marketing-image` 专家用于生成新图片，注意保留提示词内容
4. **不要展示图片**：前端会自动展示图片，你不需要在回复中用 Markdown 语法插入图片
