# 数字人视频流程

## 适用场景

用户需要生成数字人视频，例如：
- "让这张图里的人说这段话"
- "用这张人物图片生成口播视频"
- "生成数字人口播"
- "用图片和文案做一个会说话的人物视频"

## 不适用场景

以下需求不是数字人视频，不能走本流程：
- 视频克隆、视频复刻、参考视频复刻
- 替换已有视频里的模特或人物
- 基于参考视频保持动作/镜头/场景并换人物

这些需求必须加载视频克隆 SOP，并调用 `marketing-video` 专家处理。

## 流程步骤

### 步骤1：收集数字人素材

如果用户已经提供对应信息则跳过；缺失时使用 `ask_user` 收集：

1. **人物图片（必填）**：用于生成数字人的静态人物图
2. **口播文本（必填）**：数字人需要说的话
3. **参考音频（可选）**：如果用户上传了音频，必须透传真实音频 URL；否则由专家调用 `generate_reference_audio` 按文本生成通用参考音频
4. **画面比例/时长**：从用户视频偏好中读取，用户未设置时由专家按工具默认值处理

### 步骤2：委托给数字人专家

收集完需求后，必须调用 `call_agent` 将任务委托给数字人专家 `digital-human-creator`。

```
call_agent(
    AgentName="digital-human-creator",
    task_description="请为用户生成数字人视频，需求如下：
    - 人物图片：[图片URL]
    - 口播文本：[用户提供的文案]
    - 参考音频：[音频URL；如无音频，请调用 generate_reference_audio 生成]
    - 比例/时长：[从用户视频偏好中读取]

    这是数字人口播任务：让图片中的人物开口说话并生成口型同步视频。禁止按视频克隆处理。",
    user_input="[用户的原始描述]"
)
```

委托要求：
- 如果用户已经上传音频，PM 必须在任务描述中明确写出真实音频 URL，而不是只写"用户已提供参考音频"
- 如果用户没有上传音频，专家必须调用 `generate_reference_audio` 生成通用参考音频
- 专家必须实际调用 `generate_digital_human` 提交任务；严禁只返回"参数已准备完成，可以提交"
- 专家返回结果必须包含非空 `project_ids`，否则视为未提交成功

### 步骤3：确认专家返回并结束

专家 `digital-human-creator` 返回工作总结后，包含：
- `project_ids`：已提交的视频生成任务 ID
- 使用的图片、文本、音频信息
- 或者失败原因

专家返回工作总结即表示任务已提交；前端会自动轮询视频生成进度并展示结果。

## 注意事项

1. **必须调用数字人专家**：数字人视频只委托给 `digital-human-creator`
2. **不要调用营销视频专家**：`marketing-video` 用于普通视频生成和视频克隆，不负责数字人口播
3. **不要混淆视频克隆**：如果用户提供参考视频并要求复刻/替换模特，应切换到视频克隆 SOP
4. **不要重复报告进度**：专家返回工作总结后，不要再说"视频正在生成中，请稍等"
